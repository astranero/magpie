Quick win/loss guide

MAGPIE WINS WHEN
  • Privacy is non-negotiable — regulated data, proprietary research, FERPA-adjacent, or "nothing leaves my machine".
  • User lives in the browser and wants to capture as they read (pages, YouTube, PDFs) instead of uploading files.
  • Citations must be verifiable to the exact passage — audit-grade reports.
  • They want their own model (local Ollama, or a specific frontier model via key).
  • One permanent, exportable, cross-queryable library beats siloed notebooks.
NOTEBOOKLM WINS WHEN
  • Audio/video overviews are the draw (students, commuter learning, content teams).
  • Team collaboration and sharing matter.
  • Zero-setup, polished, free-tier product is required — no API keys, no extension install.
  • They need agentic code execution over sources (data analysis in-notebook).
  • Google Workspace shop with Drive-native workflows.

Profile

  • What it is: Google's source-grounded AI research notebook — upload/import sources, chat with citations, generate study/media artifacts.
  • Positioning: "Do your best research" — understanding & learning from YOUR sources, now expanding agentic (Gemini 3.5 + Antigravity, cloud compute).
  • Pricing (post-I/O May 2026): Free · Plus $4.99/mo (dropped from $7.99 in June) · Pro $19.99 ($9.99 students) · Ultra $99.99 / $200.
  • Model: Gemini 3.5, 1M-token context on every tier (limits scale, intelligence doesn't).

Recent releases (last ~90 days)

  • Jun 8: Agentic upgrade — Gemini 3.5 + Antigravity, per-notebook secure cloud computer (code execution), 100+ skills, agentic source discovery, PDF/DOCX/XLSX/PPTX/chart exports. Ultra + Workspace business first; no expansion schedule announced.
  • Jun 9: Plus price cut $7.99 → $4.99/mo.
  • May (I/O): Auto Drive sync (living documents), Ultra split into $99.99/$200 SKUs, Workspace Studio "Ask NotebookLM" automation step.
  • Apr: Source auto-labeling, bulk sharing, flashcard mastery tracking.

Strategy read: Google is pushing NotebookLM from "study tool" toward an agentic research workspace — but gating the new capabilities behind Ultra/Workspace, which widens the price gap Magpie can exploit.

Where they win

  • Media generation — audio (80 langs), video, cinematic (Veo 3), slides, quizzes. Nothing comparable in Magpie. Counter: "those are presentation formats — is the underlying research verifiable?"
  • Polish + zero setup — sign in and go. Counter: setup is one extension + one key; the trade is ownership.
  • Cloud code execution — real data analysis over sources. Counter: Ultra-only ($99.99+); most users never see it.
  • Collaboration/sharing — Plus+ tiers, Workspace integration. Counter: single-researcher workflows don't need it; markdown export is portable.

Where you win

  • Local-first privacy — their docs live on Google Cloud, no offline mode, "walled garden" is a recurring review complaint. Magpie: everything on-device, verifiable in code.
  • Siloed notebooks — top G2 complaint: no cross-notebook query, export friction. Magpie: one global library, everything queryable, plain markdown export.
  • Verification pipeline — Magpie's reranker faithfulness pass + quality audit; NotebookLM warns it "may hallucinate" with no verification layer.
  • Browser-native capture — Magpie captures the page/video/PDF you're reading; NotebookLM requires upload/import.
  • Hard caps — 50 sources/notebook free, chat quotas daily. Magpie has no per-notebook caps.

Probe the three known walls: source caps, notebook silos, export. "What happens when a project outgrows one notebook? Can you search across all of them? Can you take your library with you?" Magpie's answer to all three is structural, not a bigger quota.

Objection handling

  • "It's Google — it'll always be better." → Different axis: Google optimizes for cloud scale and media; Magpie optimizes for ownership, verifiability, and browser-native capture. Google structurally can't do local-first.
  • "NotebookLM is free." → Free tier = 50 sources, 50 chats/day, 10 deep-research/month. Magpie is free software; with local Ollama the entire stack costs $0 and stays offline.
  • "We need audio overviews." → Concede honestly — that's NotebookLM's killer feature. Position Magpie as the research layer; export markdown wherever you like (including into NotebookLM for the podcast).
  • "Setup looks technical." → One extension install + paste one API key. Fair trade for owning the data; Ollama path needs no key at all.

What Magpie is

Local-first research assistant living in the browser: capture anything you read (pages, YouTube transcripts, PDFs, images), build a permanent on-device library with local embeddings, chat with it with chunk-level verified citations, and run agentic deep research — with any model you choose.

Recent releases (this cycle)

  • SOTA research pipeline: perspective-guided planning, outline–search co-evolution, section-scoped synthesis with capstone (exec summary + contradictions + verdict) — reports 2-3× deeper.
  • Verification layer: reranker faithfulness pass on every citation + weighted LLM-judge audit (depth/evidence/honesty rubric) + honest "Remaining questions" display.
  • Source quality ranking: citation counts, venue prestige, tiered domain authority (w3/arXiv/ACM → MDN/NN/g → .gov/.edu), retraction filtering.
  • Security hardening: prompt-injection sandwich defense, MCP https-only policy, Jina proxy opt-out — all three tracked trust decisions closed.
  • Crash-proof jobs: research runs survive browser/worker restarts and resume mid-synthesis; per-tab sidepanel; live progress for every command.
  • In flight: /academic mode — papers-only research (S2/CrossRef/arXiv + BibTeX) that refuses to pad with blogs.

Key differentiators (proof points)

  • Nothing leaves the device unless you choose the endpoint — inspectable in the code (SECURITY.md documents every egress).
  • Click any citation → the exact highlighted source passage. Machine-verified before the report ships.
  • One library, every project — global Lore, cross-workspace retrieval, plain-markdown export, optional Drive/local-folder sync.
  • BYO intelligence — local Ollama to frontier models; the tool doesn't gatekeep capability behind tiers.

Honest gaps to own: no audio/video artifacts, single-user, pre-release polish, scraping-dependent extraction on hostile sites.
